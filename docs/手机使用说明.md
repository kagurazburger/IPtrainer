# 手机上使用 Memory Trainer（同步 / 读取本地 JSON）

有三种方式在手机上使用本 App，并用到「本地」或「同步」的 JSON 卡牌数据。

---

## 方式一：手机直接导入 JSON（推荐，不依赖电脑）

适合：JSON 文件在手机里（微信/QQ/文件管理器/网盘 等）。

1. **在手机上打开 App**
   - 开发时：电脑运行 `npm run dev`，并加上 `--host`（见下方「手机访问开发版」）。
   - 或：把前端部署到服务器后，用手机浏览器打开该网址。

2. **导入 JSON**
   - 进入 Collections 页面，点击 **「从 JSON 导入」**。
   - 从手机里选择你的 `.json` 文件（卡牌格式：`{ "title": "可选", "cards": [...] }` 或直接 `[...]`）。
   - 导入后会生成新卡组，可正常练习；若已登录云端，会自动尝试保存并同步。

**特点**：不需要电脑、不需要后端，数据在手机本地；需要多端同步时可登录云端。

---

## 方式二：手机读取电脑上的 JSON（同一 WiFi，同步本机资料）

适合：JSON 放在电脑的某个文件夹（如 `decks`），希望手机和电脑「共用同一份资料」。

### 1. 电脑端

1. **配置后端要读取的文件夹**  
   编辑 `backend/config.json`，设置 `default_deck_paths`，例如：
   ```json
   "default_deck_paths": ["E:\\AppDev\\live-memory-trainer-v8.0\\decks"]
   ```
   把 JSON 卡牌文件放在该目录（或子目录）下。

2. **启动后端**  
   在电脑上运行 `start.bat`（或 `python backend/main.py`），确保后端已启动。

3. **前端要连到电脑的 IP**  
   - 在项目根目录创建或编辑 `.env.local`，写入（把 IP 改成你电脑在本局域网的 IP）：
     ```
     VITE_API_URL=http://192.168.1.27:8000
     ```
   - 在 Windows 上查本机 IP：`ipconfig`，看「IPv4 地址」（如 192.168.1.27）。

### 2. 手机端

1. **手机和电脑连同一个 WiFi。**

2. **在手机上打开前端页面**  
   - 开发时：电脑运行 `npm run dev -- --host`，终端会显示类似 `Local: http://192.168.1.27:5173/`，手机浏览器访问 `http://192.168.1.27:5173`（IP 改为你电脑的）。  
   - 或：前端已部署到服务器时，用手机打开该网址即可。

3. **使用**  
   - 未登录时：App 会请求电脑后端的 `/api/default-decks`，即**同步读取电脑上配置的本地 JSON**，卡组会显示在列表中。  
   - 已登录时：会先拉取云端数据，再尝试合并电脑上的本地卡组（若手机访问不到电脑后端，则仅显示云端数据）。

**特点**：手机和电脑「共用」电脑上那份 JSON 资料；电脑关掉或不在同一网络时，手机只能看到之前同步到云端或本机已导入的数据。

---

## 方式三：导出 / 从备份恢复（不依赖云端，推荐）

适合：云端不好用时，用「导出全部」把卡组备份成一份 JSON，传到手机后「从备份恢复」。

1. **电脑上**：在 Collections 页面点击 **「导出全部」**，会下载一份 `memory-trainer-backup-日期.json`（包含当前全部卡组）。
2. **传到手机**：用微信/QQ/网盘/邮件等把该文件发到手机。
3. **手机上**：打开 App，点击 **「从备份恢复」**，选择刚传过来的备份 JSON，即可恢复全部卡组。

**特点**：不依赖云端服务，数据在自己手里；多端同步靠手动传文件。

---

## 方式四：云端同步（电脑 → 云端 → 手机）

适合：在电脑上已经导入/生成了卡组，希望手机上也有一份，且不依赖电脑开机。

1. **电脑上**：导入 JSON 或创建卡组后，点击右上角头像 **登录/生成同步 Key**，登录后卡组会自动同步到云端。
2. **手机上**：用同一网络打开 App（方式一或二的任一方式），点击头像 **用同一 Key 登录**。
3. 登录成功后，手机会拉取云端数据，即可看到在电脑上同步过的卡组。

**说明**：若云端同步失败，顶部会提示「云端同步失败，可用『导出全部』备份到本地」，可点云图标重试，或改用方式三。

---

## 小结

| 需求                     | 建议方式 |
|--------------------------|----------|
| JSON 在手机里，只想在手机用 | **方式一**：从 JSON 导入 |
| JSON 在电脑文件夹，手机要读同一份 | **方式二**：同 WiFi + 后端 + VITE_API_URL 指向电脑 |
| 电脑已有卡组，希望手机能看（不依赖云端） | **方式三**：导出全部 → 传文件到手机 → 从备份恢复 |
| 电脑已有卡组，希望自动同步到手机 | **方式四**：电脑同步到云端，手机登录同一账号 |

**开发时让手机访问本机前端**：在项目根目录执行  
`npm run dev -- --host`  
然后用手机浏览器打开终端里显示的 `http://你的电脑IP:5173`。
