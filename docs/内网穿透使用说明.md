# 内网穿透使用说明

## 快速开始

### 方法一：使用 localtunnel（推荐，最简单）

#### 1. 首次设置（只需一次）
```bash
# 运行设置脚本
setup_tunnel.bat
```

#### 2. 启动应用和内网穿透
```bash
# 步骤1：启动应用
start.bat

# 步骤2：在新窗口启动内网穿透
tunnel.bat
```

#### 3. 获取公共URL
运行 `tunnel.bat` 后，会打开两个窗口，每个窗口会显示一个公共URL，例如：
- 前端：`https://random-name-123.loca.lt`
- 后端：`https://random-name-456.loca.lt`

#### 4. 配置前端使用后端URL
需要修改前端代码，让它使用后端的内网穿透URL。

**方法A：使用环境变量（推荐）**
1. 创建 `.env.local` 文件（如果不存在）
2. 添加：
   ```
   VITE_API_URL=https://你的后端URL.loca.lt
   ```
3. 重启前端服务

**方法B：修改代码**
编辑 `services/localService.ts`，将 `API_BASE_URL` 改为后端的内网穿透URL。

#### 5. 在手机上访问
在手机浏览器中打开前端URL（例如：`https://random-name-123.loca.lt`）

## 方法二：使用 ngrok（需要注册，更稳定）

### 1. 注册 ngrok 账号
访问：https://ngrok.com/
注册并获取 authtoken

### 2. 安装 ngrok
下载：https://ngrok.com/download
或使用包管理器：
```bash
# Windows (使用 Chocolatey)
choco install ngrok

# 或直接下载解压
```

### 3. 配置 ngrok
```bash
ngrok config add-authtoken 你的authtoken
```

### 4. 启动隧道
```bash
# 启动前端隧道
ngrok http 5173

# 在另一个终端启动后端隧道
ngrok http 8000
```

### 5. 获取URL
ngrok 会显示类似：
```
Forwarding  https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:5173
```

### 6. 配置前端
同样需要修改前端代码使用后端URL。

## 方法三：使用 serveo（无需安装，但可能不稳定）

### 1. 使用 SSH 隧道
```bash
# 前端
ssh -R 80:localhost:5173 serveo.net

# 后端（新终端）
ssh -R 80:localhost:8000 serveo.net
```

### 2. 获取URL
serveo 会显示分配的URL。

## 完整配置示例

### 使用 localtunnel 的完整流程

1. **首次设置**
   ```bash
   setup_tunnel.bat
   ```

2. **启动应用**
   ```bash
   start.bat
   ```
   等待应用完全启动（看到 "Backend: http://localhost:8000" 和 "Frontend: http://localhost:5173"）

3. **启动内网穿透**
   ```bash
   tunnel.bat
   ```
   会打开两个窗口，分别显示前端和后端的公共URL

4. **配置前端使用后端URL**
   
   创建或编辑 `.env.local` 文件：
   ```
   VITE_API_URL=https://你的后端URL.loca.lt
   ```
   
   或者直接修改 `services/localService.ts`：
   ```typescript
   const API_BASE_URL = 'https://你的后端URL.loca.lt';
   ```

5. **重启前端**
   停止前端（Ctrl+C），然后重新运行：
   ```bash
   npm run dev
   ```

6. **在手机上访问**
   打开手机浏览器，访问前端URL（例如：`https://random-name-123.loca.lt`）

## 注意事项

### 1. 安全性
- localtunnel 的免费版本会显示警告页面，需要点击"Continue"才能访问
- 建议使用 ngrok 的付费版本或自建服务器以获得更好的安全性

### 2. URL 变化
- localtunnel 每次启动URL都会变化
- ngrok 免费版URL也会变化
- 如果需要固定URL，需要使用 ngrok 付费版或自建服务器

### 3. 性能
- 内网穿透会有一定的延迟
- 建议在局域网内使用时直接使用局域网IP（如果手机和电脑在同一网络）

### 4. 局域网访问（更简单的方法）
如果手机和电脑在同一WiFi网络，可以直接使用局域网IP：

1. 查看电脑IP地址：
   ```bash
   ipconfig
   ```
   找到 IPv4 地址，例如：`192.168.1.100`

2. 在手机上访问：
   - 前端：`http://192.168.1.100:5173`
   - 后端：`http://192.168.1.100:8000`

3. 修改前端配置：
   创建 `.env.local`：
   ```
   VITE_API_URL=http://192.168.1.100:8000
   ```

## 故障排除

### 问题1：localtunnel 连接失败
- 检查防火墙设置
- 尝试使用不同的端口
- 使用 ngrok 作为替代

### 问题2：手机无法访问
- 确认内网穿透服务正在运行
- 检查URL是否正确
- 尝试在电脑浏览器中访问公共URL测试

### 问题3：API请求失败
- 确认后端URL配置正确
- 检查后端服务是否正常运行
- 查看浏览器控制台的错误信息

### 问题4：CORS错误
- 确认后端已配置CORS（代码中已配置）
- 检查前端请求的URL是否正确

## 推荐方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **局域网IP** | 最快、最简单、无延迟 | 需要同一网络 | 手机和电脑在同一WiFi |
| **localtunnel** | 免费、无需注册、简单 | URL会变化、可能有警告页 | 快速测试、临时使用 |
| **ngrok** | 稳定、功能丰富 | 需要注册、免费版有限制 | 长期使用、需要固定URL |
| **自建服务器** | 完全控制、固定URL | 需要服务器、配置复杂 | 生产环境 |
