# 手机访问快速指南

## 🚀 最快方法：局域网访问（推荐）

**适用场景**：手机和电脑在同一WiFi网络

### 一键配置
```bash
# 运行自动配置脚本
get_ip_and_configure.bat
```

### 手动配置
1. 查看IP：`ipconfig`（找到IPv4地址，例如：192.168.1.100）
2. 配置前端：创建 `.env.local`，添加 `VITE_API_URL=http://你的IP:8000`
3. 重启前端：停止后运行 `npm run dev`
4. 手机访问：`http://你的IP:5173`

**优点**：最快、最简单、无延迟、免费

---

## 🌐 方法二：内网穿透（localtunnel）

**适用场景**：手机和电脑不在同一网络，或需要外网访问

### 快速开始
```bash
# 1. 首次设置（只需一次）
setup_tunnel.bat

# 2. 启动应用
start.bat

# 3. 启动内网穿透
tunnel.bat

# 4. 配置前端使用后端URL
configure_tunnel.bat https://你的后端URL.loca.lt
```

### 详细步骤
1. **设置**：运行 `setup_tunnel.bat`（安装localtunnel）
2. **启动应用**：运行 `start.bat`
3. **启动隧道**：运行 `tunnel.bat`，会显示两个URL
4. **配置前端**：运行 `configure_tunnel.bat 后端URL`
5. **重启前端**：停止后运行 `npm run dev`
6. **手机访问**：使用前端URL

**优点**：免费、无需注册、简单  
**缺点**：URL会变化、可能有警告页

## 📱 完整使用流程

### 场景A：同一WiFi（最简单）

```bash
# 步骤1：自动配置
get_ip_and_configure.bat

# 步骤2：启动应用
start.bat

# 步骤3：在手机浏览器访问显示的URL
```

### 场景B：不同网络（需要内网穿透）

```bash
# 步骤1：设置内网穿透
setup_tunnel.bat

# 步骤2：启动应用
start.bat

# 步骤3：启动隧道
tunnel.bat
# 复制显示的后端URL

# 步骤4：配置前端
configure_tunnel.bat https://复制的后端URL

# 步骤5：重启前端（在start.bat窗口按Ctrl+C，然后运行npm run dev）

# 步骤6：在手机浏览器访问前端URL
```

## 🔧 配置文件说明

### .env.local
前端API地址配置，创建在项目根目录：
```
VITE_API_URL=http://192.168.1.100:8000
```
或
```
VITE_API_URL=https://xxxxx.loca.lt
```

## ⚠️ 常见问题

### 问题1：手机无法访问
- ✅ 检查防火墙是否允许5173和8000端口
- ✅ 确认手机和电脑在同一网络（局域网访问）
- ✅ 确认内网穿透服务正在运行（内网穿透）

### 问题2：API请求失败
- ✅ 检查 `.env.local` 中的URL是否正确
- ✅ 确认后端服务正在运行（http://localhost:8000）
- ✅ 重启前端服务使配置生效

### 问题3：localtunnel连接失败
- ✅ 检查网络连接
- ✅ 尝试使用局域网访问方法
- ✅ 考虑使用ngrok（更稳定）

## 🎯 推荐方案

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 手机和电脑在同一WiFi | **局域网访问** | 最快、最简单、无延迟 |
| 手机和电脑在不同网络 | **内网穿透** | 唯一可行方案 |
| 需要固定URL | **ngrok付费版** | 提供固定域名 |
| 临时测试 | **localtunnel** | 免费、快速 |
